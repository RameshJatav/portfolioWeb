Node.js Interview Questions & Answers

1. What is Node.js?

Node.js is a runtime environment built on Chrome's V8 JavaScript engine. It allows you to run JavaScript code server-side, enabling developers to use JavaScript for both client-side and server-side development. It is designed for building scalable network applications, especially web servers and APIs.

2. What is the Event Loop in Node.js?

The event loop is a key concept in Node.js. Node.js is single-threaded, meaning it can handle one operation at a time. However, it can handle asynchronous operations (like reading files or handling HTTP requests) in the background without blocking the main thread. The event loop is responsible for managing these asynchronous tasks and ensures that non-blocking tasks are executed when they are completed.

3. What is the difference between synchronous and asynchronous code in Node.js?

Synchronous code: Executes one operation at a time, blocking the next task until the current one is completed.
Asynchronous code: Executes operations in the background and doesn't block the execution of the next task. Node.js heavily uses asynchronous code to handle I/O operations, making it more scalable and efficient.

4. What are some important modules in Node.js?

Some of the core modules in Node.js include:
- HTTP: To create HTTP servers and clients.
- fs (File System): To interact with the file system (read/write files).
- path: For working with file and directory paths.
- events: To handle events and event-driven programming.
- util: For utility functions.
- os: To get information about the operating system.

5. What is NPM (Node Package Manager)?

NPM is the default package manager for Node.js. It allows you to install, share, and manage dependencies (libraries and packages) in your Node.js projects. It provides access to a large ecosystem of open-source libraries that can be used in your applications.

6. What is a callback function in Node.js?

A callback function is a function passed as an argument to another function, which is executed after the completion of a certain operation. Node.js relies heavily on callback functions for handling asynchronous code. For example, in the fs.readFile method, a callback is provided to handle the data once the file is read.

7. What is the difference between require and import in Node.js?

- require is the older way of importing modules in Node.js and is used in CommonJS modules.
- import is part of ECMAScript modules (ESM) and allows you to import modules using the import keyword. As of Node.js v14 and above, you can use import if your code is using ESM.

8. What is a Promise in Node.js?

A Promise is an object that represents the eventual completion or failure of an asynchronous operation. It provides a cleaner way to handle asynchronous code, avoiding callback hell. A Promise has three states:
- Pending: The operation is still ongoing.
- Resolved (Fulfilled): The operation completed successfully.
- Rejected: The operation failed.
You can handle the results using .then() for success or .catch() for failure.

9. What is the difference between setTimeout() and setInterval() in Node.js?

- setTimeout(): Executes a function once after a specified delay.
- setInterval(): Executes a function repeatedly at a specified interval.

10. What is the purpose of the buffer module in Node.js?

The buffer module in Node.js is used to handle binary data directly in memory, outside the JavaScript string encoding. Buffers are useful when working with I/O operations, like reading from files or dealing with network streams.

11. Explain the difference between process.nextTick() and setImmediate() in Node.js.

- process.nextTick(): Defers the execution of a function to the next iteration of the event loop but before any I/O tasks are executed.
- setImmediate(): Defers the execution of a function until the next cycle of the event loop, after the current event loop has completed.

12. What is the use of the cluster module in Node.js?

The cluster module allows you to create child processes that run simultaneously and share the same server port. This helps in utilizing multi-core systems efficiently. It allows Node.js to scale and handle more requests by distributing the load among multiple processes.

13. What is CORS, and how do you handle it in Node.js?

CORS (Cross-Origin Resource Sharing) is a mechanism that allows web browsers to make requests to domains other than the one from which the web page was served. In Node.js, you can handle CORS by using the cors package, which is commonly added to the server as middleware to allow or restrict cross-origin requests.

14. Explain middleware in Express.js.

Middleware functions in Express.js are functions that have access to the request, response, and the next middleware function in the application's request-response cycle. Middleware is often used to modify the request/response objects, log information, handle authentication, or even handle errors.

15. What is the role of the req and res objects in Express.js?

- req (Request object): Represents the HTTP request and contains information about the request such as query parameters, body, and headers.
- res (Response object): Represents the HTTP response and is used to send a response back to the client. Methods like res.send(), res.json(), and res.status() are commonly used to send responses.
